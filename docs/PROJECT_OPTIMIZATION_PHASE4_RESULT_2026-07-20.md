## 实施内容

### Settings

原 `backend/api/settings.py` 同时承载请求模型、环境文件处理、连接探测、集合重建辅助逻辑和 API 编排。现拆分为：

- `api/settings_models.py`：Pydantic 请求/响应模型与共享校验器。
- `api/settings_env.py`：`.env` 读写、布尔值解析和密钥掩码。
- `api/settings_connection.py`：LLM/Embedding 连接探测。
- `api/settings_rebuild.py`：重建预检、稳定文档键、BM25 双缓冲清理与原子切换。
- `api/settings.py`：保留路由、业务编排以及旧导入兼容层。

`settings.py` 从约 1006 行降至 742 行，减少约 26%。拆分过程中还删除了两个确认无调用的旧辅助函数。

### Memory

原 `backend/memory/profile.py` 混合了画像纯函数、持久化和向量索引编排。现拆分为：

- `memory/profile_core.py`：画像评分、淘汰、扁平化、ID 解析和文本格式化纯逻辑。
- `memory/profile_store.py`：用户画像数据库读取与保存。
- `memory/profile.py`：保留画像业务编排、索引联动与旧符号兼容层。

`profile.py` 从约 545 行降至 490 行。纯逻辑和存储逻辑已经可以独立测试，且不再需要通过完整画像编排模块覆盖。
