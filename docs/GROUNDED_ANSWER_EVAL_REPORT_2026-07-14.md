# RAG 端到端答案可信度在线 A/B 评测报告

> 评测时间：2026-07-14  
> 数据集：`rag-agent-eval-v2` 2.0  
> 在线模型：DeepSeek `deepseek-v4-flash`

## 1. 评测设计

- 人工标注查询：93 条。
- 可回答：79 条；应拒答：14 条。
- 在线生成：186 次，全部成功，无接口错误。
- 检索：两组固定使用相同的生产 splitter + BM25 Top-5 结果。
- 对照组：旧式“仅依据来源回答”Prompt，不强制引用。
- 优化组：关键声明强制 `[S<n>]` 引用，并明确要求资料不足时拒答。
- Faithfulness、引用精确率和引用完整率仅在 79 条可回答查询上计算。
- 拒答准确率仅在 14 条空 qrels 查询上计算。

该设计评估的是“回答可信度优化”本身，不把检索结果变化混入 A/B。

## 2. 核心结果

| 指标 | 优化前 | 优化后 | 变化 |
|---|---:|---:|---:|
| Faithfulness | 69.55% | 69.49% | -0.06 个百分点 |
| 引用精确率 | 0.00% | 68.41% | +68.41 个百分点 |
| 引用完整率 | 0.00% | 71.32% | +71.32 个百分点 |
| 拒答准确率 | 100.00%（14/14） | 100.00%（14/14） | 持平 |
| 期望事实覆盖率（辅助指标） | 74.13% | 67.40% | -6.73 个百分点 |
| 平均生成延迟 | 42.40s | 86.65s | 旧计时包含并发排队，不可用于 A/B |

以下是旧计时观测值，仅保留用于审计，不能解释为模型服务延迟：

| 模式 | P50 | P95 |
|---|---:|---:|
| 优化前 | 40.81s | 81.36s |
| 优化后 | 86.91s | 89.12s |

## 3. 引用与忠实度分布

- 优化后 79 条可回答查询中，43 条达到 `verified`。
- 50 条引用精确率达到 100%。
- 46 条引用完整率达到 100%。
- 16 条仍完全没有有效声明级引用。
- Faithfulness 配对比较：14 条提升、17 条下降、48 条持平。
- 期望事实覆盖配对比较（76 条有事实标注）：10 条提升、16 条下降、50 条持平。

## 4. 结论

### 已确认收益

1. 强制来源编号显著补齐了项目原本缺失的引用能力。
2. 引用精确率达到 68.41%，说明多数引用能支持对应声明，但距离生产门槛仍有明显差距。
3. 引用完整率达到 71.32%，多数关键声明已经带引用。
4. 拒答准确率保持 100%，没有因引用优化破坏无答案场景。

### 当前回归

1. Faithfulness 没有显著提升，当前收益主要是“可追溯”，不是事实支持率提升。
2. 优化 Prompt 对“部分可回答”的复杂问题过度保守，导致期望事实覆盖下降 6.73 个百分点。
3. 原评测在获取并发槽位前开始计时，优化组因执行顺序包含更多本地排队时间；延迟差异无效。V2 运行器已把计时起点移到并发槽位内部。
4. 16 条可回答查询仍没有有效声明级引用，距离稳定上线门槛尚有差距。

## 5. 测量修正说明

第一次汇总时，验证器会把“根据检索资料……”开头的整句误判为元话语并丢弃，造成优化组指标虚低；拒答检测也没有识别“无法提供”这种明确拒绝表达。V2 又补充了“事实。 [S1]”到“事实 [S1]。”的声明绑定归一化，消除了引用被错误绑定到下一句的虚高。本报告已使用最终口径对同一批 186 条已保存答案重新评分，没有再次调用在线模型。

## 6. 上线判断与下一轮建议

当前版本不建议直接把严格 Prompt 作为默认策略，建议先做第二轮小步优化：

1. 将“资料不足就整体拒答”改成“先回答有依据的部分，再明确列出无法确认的部分”。
2. 要求每个列表项分别引用，避免一个 `[S1]` 放在长段末尾造成声明覆盖失败。
3. 对生成长度设上限，去掉“根据检索资料”等固定前缀，降低延迟和验证噪声。
4. 目标门槛：引用精确率 ≥ 85%、引用完整率 ≥ 90%、Faithfulness 不低于对照组、事实覆盖下降不超过 2 个百分点。
5. 调整 Prompt 后只在固定 dev 子集选择版本，最后在 93 条全集上复验一次。
