## 2. V3 策略

### 2.1 证据内并列（V3.1 收敛）

- 允许把来源明确写出的事实结构化并列，但不从并列事实生成新结论。
- 禁止新增共同点、差异、因果、优劣、最高级、适用场景和操作步骤。
- 比较题使用“A 的资料事实 / B 的资料事实 / 无法确认的比较维度”结构。
- 只有来源原文明确给出比较结论时才能复述该结论。
- 缺少一方资料时回答已有一方并说明限制，不得整体拒答。
- 每个列表项只包含一个来源可完整支持的事实并独立引用。

### 2.2 覆盖优先

- “什么是、有哪些、概览、详细说明”覆盖全部直接相关且互不重复的定义、类别、代表例子、关键数字和限制。
- 将默认上限从 300 字调整为 500 字或 6 个列表项。
- 简洁只删除重复和无关信息，不省略直接相关事实。

### 2.3 答案级三态与完成度

- `full`：资料完整支持所问结论。
- `partial`：先回答有证据部分，再明确无法比较或确认的部分。
- `none`：资料没有所问关系、公式或适用场景，或问题缺少上下文。
- 回答含限制语但已有受支持声明时，不再判定为整体拒答。

本轮审计将 Bomba 米比较、Django ORM 与 FastAPI SQLAlchemy 体验、光伏与风电成本标为部分可答；将 MinMax/Z-score 适用场景和 F1 计算公式标为不可答。检索相关性标注保持不变。
