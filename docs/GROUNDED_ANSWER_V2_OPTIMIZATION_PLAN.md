# Grounded Answer V2 优化方案

> 日期：2026-07-14  
> 输入：93 条人工标注查询、186 次真实在线生成结果

## 1. 问题基线

第一轮在线 A/B 经声明级引用位置归一化后的主要结果：引用精确率 68.41%、引用完整率 71.32%、Faithfulness 69.49%、拒答准确率 100%，但期望事实覆盖率从 74.13% 降至 67.40%。

失败样本表明，第一版 Prompt 把“不能完整回答”错误地等同于“应该整体拒答”。34 条有相关 qrels 的查询出现拒答表达，其中不少问题实际上可以回答一部分。另一个测量问题是评测脚本在获取并发槽位之前就启动计时，导致排队时间被算入模型延迟，优化组总是排在对照组之后，原延迟对比不能用于 SLA 判断。

## 2. 优化目标

| 指标 | V2 目标 |
|---|---:|
| 引用精确率 | ≥ 85% |
| 引用完整率 | ≥ 90% |
| Faithfulness | 不低于对照组 |
| 期望事实覆盖率 | 相对对照组下降 ≤ 2 个百分点 |
| 拒答准确率 | ≥ 95% |

## 3. 策略设计

### 3.1 部分回答优先

- 来源能支持任何一部分时，先完整回答有依据的部分。
- 对来源不能支持的部分，单独写“无法确认：……”。
- 只有来源对问题完全没有直接支持时，才整体拒答。
- 禁止因为问题包含一个不可回答的子问题而丢弃其他可回答事实。

### 3.2 声明级引用

- 每个事实句或列表项独立引用。
- 引用放在声明末尾、句号之前：`事实 [S1]。`
- 不使用段尾单个引用覆盖多个独立声明。
- 不输出“根据检索资料”等固定前缀。

### 3.3 输出收敛

- 默认控制在 300 个中文字符以内；用户明确要求详细说明时例外。
- 删除重复复述和无信息量的开场语。
- 优先直接答案，再给必要限制。

### 3.4 评测口径修正

- 并发槽位获取成功后再开始模型计时，排除本地队列等待。
- 验证器兼容引用写在句号后的模型输出，并在切句前归一化。
- A/B 两组继续固定相同检索结果，避免召回差异污染答案策略比较。

## 4. 验收流程

1. 运行 Prompt 契约、verifier、Agent/API 专项测试。
2. 在固定 dev 子集复测 V2 Prompt。
3. 达到门槛后，运行全部 93 条测试集。
4. 使用 `--rescore` 对保存答案重复评分时不得再次调用模型。
5. 若事实覆盖仍下降超过 2 个百分点，不将严格策略设为默认。

## 5. 已执行改动

- 生产 system prompt 已切换为部分回答优先策略。
- 在线 A/B 的优化组 Prompt 已同步为 V2。
- 引用格式统一为句号前的声明级 `[S<n>]`。
- verifier 已兼容模型输出的句号后引用，并在切句前重新绑定。
- 延迟计时已移到并发槽位内部，排除本地队列等待。
- 新增固定 22 条失败聚焦 dev 子集，包含 19 条可回答和 3 条拒答查询。
- 新增自动质量门禁；引用、Faithfulness、事实覆盖或拒答任一不达标即失败。
- Prompt、verifier、API、Agent 与质量门禁专项测试：`24 passed`；Ruff：通过。

第一轮保存结果按最终 verifier 口径重评分后，V1 质量门禁为 `passed=false`，因此 V2 必须先通过 dev 子集才能进入全量复测。

## 6. Dev 实跑后的标签审计与重评分

V2 首次 dev 在线结果暴露出检索相关性与答案可答性被混为一谈。经逐条对照语料，qrels 新增三态答案标注：

- `full`：资料完整支持问题所需结论；
- `partial`：资料只支持部分答案，应先回答有证据部分再说明限制；
- `none`：资料虽出现相关实体，但没有所问关系或问题本身缺失指代，应拒答或澄清。

原检索相关性标注保持不变。使用已保存的 44 条真实在线答案离线重评分，无新增模型调用：

| 指标 | Control | V2 | 变化 |
|---|---:|---:|---:|
| Faithfulness | 79.49% | 91.03% | +11.54pp |
| 引用精确率 | 0.00% | 90.11% | +90.11pp |
| 引用完整率 | 0.00% | 91.03% | +91.03pp |
| 拒答准确率 | 66.67% | 100.00% | +33.33pp |
| 期望事实覆盖率 | 76.67% | 82.31% | +5.64pp |
| 答案完成准确率 | 72.73% | 95.45% | +22.72pp |

验证器排除了“无法确认”“做法未在资料中提及”等明确的证据限制语，避免将其误算为需要引用的外部事实。重评分后完整 dev 质量门禁为 `passed=true`，无违反项。
