## 10. 生产对话比较题回归修复

同日真实对话发现追问“mcp 和它有什么区别”虽然能从上一轮识别 `它 = Skill`，但最终只返回一条 MCP 定义。日志显示单条声明通过来源验证，而其余比较内容在流式校验和修复过程中被删除。

根因与修复：

1. 多次检索同一文档时，旧逻辑按文档统一限制片段数，可能只留下第一个检索主题；现按每次检索分组保留证据，再执行整轮上限控制；
2. 已裁剪来源仍残留在较早的工具消息中，模型可见但验证器不可见；现清空旧搜索消息，只在最新工具消息中提供与验证器一致的合并来源；
3. 简单追问的指代只依赖模型临时理解；现可将上一轮“Skill 是什么”确定性解析进当前查询，得到“mcp 和 skill 有什么区别”；
4. 比较题逐句流式放行会先发送单侧事实，后续无法撤回；现比较题先缓冲完整回答，普通事实题继续使用低延迟流式校验；
5. LLM 修复可能把多项部分支持内容压缩为单条高忠实度声明；现修复后再次执行问题关系完整性守卫，不完整时输出双侧确定性事实及无法确认的比较维度。
6. 缓存策略版本同步提升，部署后不会复用修复前已经缓存的单侧回答。

新增回归覆盖多检索主题保留、简单指代解析、复杂历史不误解析，以及比较题修复不得退化为单侧答案。Agent 全量测试 278/278 通过，Ruff、Mypy 和独立发布门禁通过；本次修复及重评分没有新增外部模型调用。
