### v0.2.0-dev (2026-07-24) — Agent 自主推理重构

#### 重大变更

- **Agent 放权**：删除意图分类规则层（确认词、跟进词、计算器、文档列表全部移除），工具选择交由模型 ReAct 自主决策；
- **系统提示词重写**：从"规则手册"改为"角色+能力+原则"，强调观察→判断→行动→评估的 ReAct 推理模式；
- **联网搜索升级**：KB 不足时 Agent 可自主升级到 web_search；web_search 结果纳入独立 WS 引证体系，与知识库 S 引证共存；
- **循环检测**：同工具 ≥3 次自动收敛，强制禁用工具直接生成文本回答，防止搜索死循环；
- **Safety guard 放权**：从正则替换答案改为通知式（信任模型自我修正，后验证兜底）；
- **流验证关闭**：内容直达用户，后验证作为安全网。

#### 改进

- **验证精度**：phantom citation 修复为 FORMAT_ONLY、数字匹配支持版本号前缀（3.12→3.12.0）和单位空格容错（100 MB→100MB）、`_is_mid_citation` 大小写不敏感；
- **验证重试**：从硬编码指令改为反馈式（告知具体失败原因，允许重试时调用 web_search）；
- **数据一致性**：索引失败同时清理 Qdrant + BM25、delete_document 和 clear_all 改为 SQLite 优先原子策略、去重保留原始来源不再替换 doc_id；
- **配置**：rrf_k 添加 ≥0 验证、stream verify 默认关闭；
- **前端**：UploadZone 改用 useRef 防止重复渲染产生多个文件选择器；
- **工具**：添加孤儿向量清理脚本 `backend/cleanup_orphans.py`、检索超时返回错误而非空结果；
- **上下文**：修复重试时历史摘要丢失和内存提取跳过的问题。
- 统一 README、环境变量模板与当前默认配置；
- 可选 OCR 与 Reranker 不再阻塞核心服务启动。

### 安全

- Refresh Token 仅保存在 HttpOnly Cookie；
- 修改密码后旧会话令牌失效；
- 默认仅监听本地回环地址；
- Qdrant 默认不向宿主机公开端口；
- 备份恢复增加路径、文件类型、容量和完整性检查。

## 0.1.0-beta - 待发布

首个公开 Beta 候选版本。

### 主要能力

- PDF、DOCX、TXT、Markdown、CSV 和 XLSX 文档入库；
- Qdrant 语义检索与 SQLite BM25 关键词检索；
- 查询分类、消歧、RRF 融合、去重和可选精排；
- ReAct 工具循环与 SSE 流式回答；
- 引用精确性、召回、数字和事实覆盖校验；
- 用户认证、权限检查、密码修改和登录恢复；
- 可恢复后台任务、数据库迁移、备份和恢复；
- 健康检查、依赖状态、指标和结构化日志；
- Docker Compose 部署以及可选 OCR、Reranker。

### 已知限制

- 当前主要面向单租户或小团队，不提供完整企业级多租户隔离；
- SQLite、本地文件和本地 Qdrant 路径不适合直接水平扩展；
- 更换 Embedding 模型可能需要重建向量集合；
- 完整问答依赖外部 OpenAI 兼容 LLM 和 Embedding 服务；
- OCR、Reranker 首次使用可能下载模型并消耗额外内存；
- Beta 版本尚未完成所有生产环境、监管场景和高并发验证。

### 升级与回退

- 升级前备份数据目录和数据库；
- 启动时由 Alembic 执行数据库迁移，并在迁移前创建快照；
- 回退代码前应恢复与旧版本兼容的数据库快照；
- 不要让旧代码直接访问已经完成不兼容迁移的新数据库。
