# 安全策略

## 支持范围

| 版本 | 状态 |
|---|---|
| `v0.1.x-beta` | 支持最新版本 |
| 更早的开发快照 | 不提供安全维护承诺 |

项目处于 Beta 阶段。安全修复原则上只进入最新版本，不承诺为旧版本长期回移补丁。

## 私下报告漏洞

请勿在公开 Issue、讨论区、Pull Request 或日志中披露漏洞细节、密钥和可利用样例。

仓库公开后，请优先通过 GitHub 仓库的 **Security → Advisories → New draft advisory** 私下提交报告。如果该入口不可用，请只提交一个不含技术细节的普通 Issue，请求维护者建立私密联系。

报告建议包含：

- 受影响版本或提交；
- 漏洞类型和影响范围；
- 最小复现步骤；
- 是否已经公开或被利用；
- 建议缓解措施；
- 是否希望在公告中署名。

请不要附带真实用户数据、真实密钥或未经授权取得的文档。

## 处理流程

维护者目标是在 5 个工作日内确认收到报告，但 Beta 项目不提供服务等级保证。确认问题后将：

1. 复现并评估影响；
2. 制定临时缓解措施；
3. 完成修复和回归测试；
4. 发布新的补丁或次要版本；
5. 在适当时机发布安全公告；
6. 经报告者同意后致谢。

已经发布的 Git 标签不会被静默修改。严重问题应通过新版本修复。

## 部署安全要求

### 认证

- 为每个环境生成独立且至少 32 字符的 `JWT_SECRET`；
- 首次部署必须设置新的 `BOOTSTRAP_ADMIN_PASSWORD`；
- 不在生产环境复用示例凭据；
- HTTPS 部署设置 `AUTH_COOKIE_SECURE=true`；
- 修改密码后验证旧 Refresh Token 已失效；
- 定期轮换外部服务密钥，并准备重新登录窗口。

### 网络

- 默认保持 `SERVER_HOST=127.0.0.1`；
- 只有在理解风险时才设置 `ALLOW_REMOTE_ACCESS=true`；
- 公网部署必须使用 TLS、反向代理、防火墙和访问频率限制；
- 不把 Qdrant 端口直接暴露到公网；
- 健康检查和监控接口不应返回密码、密钥或 Token。

### 数据与第三方服务

- 上传文档、数据库、向量索引和会话默认保存在本地；
- 使用外部 LLM、Embedding 或联网搜索时，必要的问题、文档片段或查询会发送给相应提供方；
- 上传敏感资料前应审查提供方的数据处理政策；
- 本项目不会自动对外部请求中的业务内容做匿名化；
- 备份可能包含完整数据库和密码哈希，应限制访问并按敏感数据管理；
- 备份默认不提供额外加密层，公开环境应使用受控存储。

### 可选组件

- PaddleOCR 默认本地执行，但安装和模型来源仍需进行供应链审查；
- Cross-Encoder 首次使用可能下载模型；
- 联网搜索会把查询发送给配置的搜索服务；
- 第三方插件、模型和代理服务不属于本项目的信任边界。

## 密钥泄露处理

如果密钥进入提交、日志或公开页面：

1. 立即在提供方撤销并轮换密钥；
2. 检查访问日志和影响范围；
3. 扫描当前文件与完整 Git 历史；
4. 必要时清洗历史并重新扫描；
5. 不要认为删除当前文件就已经解决历史泄露。

## 安全边界

本项目不替代专业身份提供商、数据防泄漏系统、企业密钥管理服务或完整多租户隔离。面向公网、多人或受监管数据部署前，应完成独立的威胁建模、权限审计、渗透测试、备份演练和隐私合规评估。
