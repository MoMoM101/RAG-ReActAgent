# 测试与评测数据来源说明

## 目的

本文件说明公开仓库中测试文档、检索 qrels 和质量评测数据的来源及使用边界，避免把真实用户资料、第三方受限内容或未知来源数据混入开源版本。

## 数据分类

### 基础评测文档

`tests/eval_data/` 中的支付接口、传感器、合规制度等内容是为检索测试人工编写的合成文本。名称、错误码、金额、地址、产品和组织均用于构造歧义、精确字段和跨文档干扰，不代表真实企业、服务或合同。

对应生成与维护入口包括：

- `tests/gen_eval_docs.py`；
- `tests/gen_complex_docs.py`；
- `tests/qrels_data_v2.json`；
- `tests/qrels_complex_v2.json`。

文档中使用的 `192.0.2.0/24`、`198.51.100.0/24`、`203.0.113.0/24` 等地址属于文档示例地址范围，不表示真实生产网络。

### Docker E2E 文档

`tests/e2e/fixtures/` 中的产品和退款政策由项目为容器验收人工编写，只用于验证：

- 文件上传与切片；
- Qdrant 和 BM25 索引一致性；
- 引用问答；
- 重启持久化；
- 备份恢复；
- 依赖降级。

`manifest.json` 保存文件 SHA-256、预期切片数、测试问题、预期关键词和来源文件名，用于防止夹具被静默修改。

### Grounded Answer 与检索报告

`grounded_answer_eval_*.json`、`evaluation_results*.json` 和 qrels 文件记录合成评测输入、模型输出或指标。它们不应包含真实用户会话或上传文档。

正式发布只保留被发布门禁、CI 或公开指标直接引用的必要报告；重复探针和过程报告保留在本地归档或发布资产中。

### 图片样本

`backend/test.png` 是仅包含“Hello World 12345”的合成小图片，不含个人信息或第三方素材。它当前作为历史解析样本保留，不作为产品图片或质量证明。

## 许可证与再分发

上述合成夹具、qrels、生成脚本和项目自建评测内容随本仓库按 MIT License 发布。第三方库、模型和外部 API 返回内容仍遵循各自许可证与服务条款，不因本说明获得重新授权。

## 新增数据要求

新增测试或评测数据必须同时满足：

- 来源和作者明确；
- 允许公开、修改和再分发；
- 不包含真实密钥、个人信息和业务秘密；
- 不直接复制受版权保护的长篇内容；
- 使用真实数据前完成脱敏和授权；
- 在本文件或相邻清单中记录用途与许可；
- 评测标准答案不得由同一次待评测结果反向生成。

无法确认来源或许可的数据不得进入公开分支和 Release 资产。
